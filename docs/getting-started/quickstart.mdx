# Quickstart

Find a plugin in the directory, install it, and wire it into Paperclip — usually a couple of minutes.

> You'll need Paperclip itself installed first. See [paperclip.ing](https://paperclip.ing) to get set up.

## 1. Find a plugin

Browse the full catalog on the [hub home](/), or narrow it down:

| To…                | Do this                                                            |
| ------------------ | ------------------------------------------------------------------ |
| Search by keyword  | Use the search box on the [home page](/)                           |
| Filter by category | Open a category, e.g. [`/?category=provider`](/?category=provider) |
| See the newest     | [`/?sort=newest`](/?sort=newest)                                   |

Categories group plugins by what they do — `auth`, `provider`, `tools`, `integration`, `observability`, `memory`, `other`. See [Registry Schema](/docs/registry/schema) for the full list.

## 2. Check it out

Open a plugin to see its page: what it does, its **capabilities** (the Paperclip hooks it implements), its source repository, and how many times it's been installed.

## 3. Install it

Each plugin page has an **Install** panel with the exact command for that plugin. Copy it and run it in your project — there's nothing to memorize, since the hub gives you the command per plugin.

## 4. Use it

Once installed, the plugin loads into Paperclip. For configuration and usage specifics, follow the plugin's own repository or [paperclip.ing](https://paperclip.ing) — the hub lists plugins; Paperclip runs them.
